## NPV Mechanics

### Core Formula

```
NPV = CF_0 + CF_1/(1+r)^1 + CF_2/(1+r)^2 + ... + CF_n/(1+r)^n

NPV = Sum from t=0 to n of [CF_t / (1+r)^t]
```

Where:
- CF_t = incremental after-tax cash flow in year t
- r = project-specific discount rate (hurdle rate)
- CF_0 = initial investment (typically negative)
- n = project life in years

### What Counts as an Incremental Cash Flow

An incremental cash flow is the difference between the company's cash flows with the project and without the project. Include only amounts that change because of the investment decision.

**Include**:
- Revenue directly generated by the project
- Operating costs directly caused by the project
- Capital expenditure for the project
- Working capital changes driven by the project
- Tax effects (including depreciation tax shields)
- Salvage or terminal value
- Side effects on other parts of the business (cannibalization, synergy)
- Opportunity costs (value of resources diverted from their next-best use)

**Exclude**:
- Sunk costs (already spent, unrecoverable)
- Allocated overhead that would exist regardless of the project
- Financing costs (these are captured in the discount rate, not in cash flows)
- Interest expense (already in WACC -- including it double-counts the cost of debt)

### Discounting Mechanics

Discount each cash flow back to Time 0 (the decision point).

**End-of-year convention**: Cash flows occur at the end of each year. Year 1 cash flow is discounted by 1/(1+r)^1.

**Mid-year convention** (optional, more realistic): Cash flows occur throughout the year. Use 1/(1+r)^0.5 for year 1, 1/(1+r)^1.5 for year 2, etc. This increases NPV slightly because cash flows arrive earlier on average.

**Initial investment**: Occurs at Time 0 (not discounted). If investment is phased over multiple years, discount each tranche.

### Terminal/Salvage Value

If the project has a finite life, the final year may include:

```
After-tax Salvage = Salvage Value - Tax Rate x (Salvage Value - Book Value at End)
```

- If Salvage > Book Value: taxable gain, so after-tax salvage < pre-tax salvage
- If Salvage < Book Value: tax loss, creating a tax shield
- If Salvage = Book Value: no tax impact

If the project generates cash flows indefinitely, use a terminal value (growing perpetuity):

```
Terminal Value at year n = CF_(n+1) / (r - g)
```

Then discount the terminal value back to Time 0. This approach is more common in company-level DCF than project-level analysis, but applies when the project has no natural end date.

## IRR Pitfalls

### What IRR Measures

IRR is the discount rate at which NPV equals zero. If IRR > hurdle rate, the project earns more than its cost of capital.

```
0 = CF_0 + CF_1/(1+IRR)^1 + CF_2/(1+IRR)^2 + ... + CF_n/(1+IRR)^n
```

### When IRR Works Well

- Single initial investment followed by positive cash flows (conventional cash flow pattern)
- Standalone go/no-go decision (not comparing projects of different sizes)
- Cash flows have a single sign change (negative to positive)

### When IRR Fails

**Problem 1: Multiple IRRs**

When cash flows change sign more than once (e.g., negative-positive-negative pattern from environmental cleanup costs at end of project), the IRR equation may have multiple solutions. Each mathematical root is an equally valid IRR, making interpretation ambiguous.

*Example*: Initial investment -$100, Year 1-4 cash flows +$50 each, Year 5 cleanup cost -$80. This cash flow stream (-, +, +, +, +, -) can produce two IRRs: 12% and 38%. Neither is meaningful alone.

*Solution*: Use NPV. Plot NPV vs. discount rate (the NPV profile) to visualize the relationship.

**Problem 2: Scale Problem**

IRR ignores the size of the investment. A 50% IRR on $1M creates $500K in value. A 15% IRR on $100M creates $15M in value.

*Solution*: For mutually exclusive projects, choose the project with the highest NPV, not the highest IRR. Use IRR only as a supplementary measure.

**Problem 3: Reinvestment Assumption**

IRR implicitly assumes intermediate cash flows are reinvested at the IRR itself. If the project earns 25% IRR but the company can only reinvest intermediate cash flows at 10%, the IRR overstates the true return.

*Solution*: Use Modified IRR (MIRR), which assumes reinvestment at the cost of capital:

```
MIRR: Terminal Value of inflows (compounded at WACC) / PV of outflows (at WACC)

Solve for MIRR in: PV of outflows x (1 + MIRR)^n = Terminal Value of inflows
```

**Problem 4: Timing Differences**

When comparing projects with different lives, IRR can favor shorter projects even if longer projects create more total value.

*Solution*: Use equivalent annual NPV (NPV converted to an annuity) for projects with different lives, or use NPV directly with a common time horizon.

## Project-Specific Discount Rates

### When to Adjust the Discount Rate

Use a project-specific discount rate when:
- The project is in a different industry than the parent company
- The project is in a different geographic region with different country risk
- The project has materially different business risk than the company's existing operations

The company-wide WACC is appropriate only when the project has the same risk profile as the company's average existing assets.

### Bottom-Up Beta for a Project in a Different Industry

If the project enters an industry different from the parent company:

1. **Identify comparable firms** in the project's industry (3-10 peers)
2. **Collect regression betas** for each comparable
3. **Unlever each beta** to remove capital structure effects:
   ```
   Unlevered Beta = Regression Beta / (1 + (1 - Tax Rate) x (Debt/Equity))
   ```
4. **Take the median** unlevered beta across comparables
5. **Relever** at the parent company's (or project's) target capital structure:
   ```
   Levered Beta = Unlevered Beta x (1 + (1 - Tax Rate) x (Target D/E))
   ```
6. **Compute cost of equity**: Cost of Equity = Riskfree Rate + Levered Beta x ERP

### Revenue-Weighted Country Risk Premium

For international projects, adjust the discount rate for country-specific risk:

1. **Map revenue exposure** by country or region (% of project revenue from each)
2. **Look up country risk premium** for each region (from Damodaran's country risk premium database or sovereign CDS spreads)
3. **Compute weighted average CRP**:
   ```
   Weighted CRP = Sum of (Revenue Weight_i x CRP_i) for all regions i
   ```
4. **Add to base WACC**:
   ```
   Project WACC = Base WACC (domestic) + Weighted CRP
   ```

**Example**:

| Region | Revenue % | CRP | Weighted CRP |
|--------|-----------|-----|-------------|
| North America | 40% | 0.0% | 0.00% |
| Western Europe | 15% | 0.5% | 0.08% |
| Asia | 15% | 2.5% | 0.38% |
| Central/South America | 10% | 3.0% | 0.30% |
| Africa | 5% | 4.0% | 0.20% |
| Eastern Europe | 5% | 3.5% | 0.18% |
| Australia/NZ | 5% | 0.0% | 0.00% |
| Middle East | 5% | 2.0% | 0.10% |
| **Total** | **100%** | | **1.23%** |

Base WACC: 9.0%. Project WACC: 9.0% + 1.23% = 10.23%.

### Alternative: Lambda Approach

For companies with varying exposure to country risk:

```
Cost of Equity = Riskfree Rate + Beta x Mature Market ERP + Lambda x Country Risk Premium
```

Where Lambda measures the company's or project's specific exposure to country risk (Lambda = 1 for average exposure, > 1 for higher exposure, < 1 for lower exposure). Lambda can be estimated from the fraction of revenues, assets, or production in the risky country.

## EVA Framework

### Economic Value Added (EVA)

EVA measures whether a project earns more than its cost of capital in dollar terms:

```
EVA = (ROC - WACC) x Invested Capital

Where:
ROC = After-tax Operating Income / Invested Capital
WACC = project-specific weighted average cost of capital
Invested Capital = total capital deployed in the project (book value of equity + debt used)
```

### Interpreting EVA

| EVA | ROC vs WACC | Interpretation |
|-----|-------------|---------------|
| Positive | ROC > WACC | Project creates value above its cost of capital |
| Zero | ROC = WACC | Project earns exactly its cost of capital (zero economic profit) |
| Negative | ROC < WACC | Project destroys value even if accounting profit is positive |

### Why Accounting Profit Can Mislead

A project can report positive accounting net income while still destroying economic value. Accounting profit deducts interest (cost of debt) but not the cost of equity. EVA deducts both.

**Example**: A project earns 8% ROC. The company's WACC is 10%. The project shows positive net income (because interest payments are only part of the capital cost), but EVA is negative: (8% - 10%) x $500M = -$10M/year in value destruction.

### Linking EVA to NPV

The NPV of a project can be decomposed into:

```
NPV = Invested Capital + PV of all future EVAs - Invested Capital
NPV = PV of all future EVAs
```

A positive NPV means the present value of expected future EVAs is positive -- the project is expected to earn above its cost of capital over its life.

### Steady-State vs Ramp-Up

Many projects have negative EVA in early years (ramp-up period) before reaching positive EVA at steady state. This is normal and expected. The relevant question is whether cumulative EVA (discounted) is positive over the project's entire life -- which is the same as asking whether NPV is positive.

## Sunk Costs and Opportunity Costs

### Sunk Costs

**Definition**: A sunk cost is an expenditure that has already been made and cannot be recovered regardless of whether the project proceeds.

**Rule**: Exclude sunk costs from the NPV calculation.

**Rationale**: The decision is forward-looking. Only future cash flows that change based on the invest/reject decision belong in the analysis. Money already spent is the same in both scenarios (invest or do not invest).

**Common sunk cost traps**:
- R&D already spent on product development
- Market research and feasibility studies already completed
- Consulting fees already paid
- Land already purchased (unless it has an alternative use -- then it is an opportunity cost)

**Psychological challenge**: Managers often feel obligated to proceed with a project because of money already spent. This is the sunk cost fallacy. The correct question is always: "Given where we are now, does the project create value going forward?"

### Opportunity Costs

**Definition**: An opportunity cost is the value of the next-best alternative use of a resource that is consumed by the project.

**Rule**: Include opportunity costs as a cash outflow in the NPV calculation.

**Examples**:
- Excess factory capacity used for the project could have been rented out at $5M/year. Include -$5M/year as a project cash flow.
- Land owned by the company and used for the project could be sold for $20M. Include -$20M at Time 0 (the forgone sale proceeds).
- Management time diverted from another project that would have generated $2M in annual profit. Include -$2M/year.

**Key distinction**: If a resource has no alternative use (truly idle, cannot be sold or redeployed), its opportunity cost is zero.

## Side Effects: Cannibalization and Synergy

### Cannibalization

When a new project steals sales from existing products, the lost margin on those sales is a real cost of the project.

```
Cannibalization Cost = Lost Revenue x Operating Margin on Existing Product
```

**Include** cannibalization if:
- The company's own new product causes the lost sales
- The lost sales would not have occurred anyway (competitor would not have taken them)

**Exclude** cannibalization if:
- A competitor would have launched a similar product and taken the sales regardless
- The customer segments are genuinely different

### Synergy

When a new project boosts sales of existing products (e.g., a fitness equipment line drives more subscribers to the streaming service):

```
Synergy Benefit = Incremental Revenue to Existing Business x Operating Margin
```

Include synergy as a positive cash flow in the project's analysis. Be conservative -- synergy estimates are frequently overstated.

### The Netflix Fit Example

The Netflix Fit case illustrates both:
- **Potential cannibalization**: None explicitly stated, but if fitness subscribers reduce entertainment subscriptions, that would be a negative side effect
- **Synergy**: $500M boost to core streaming revenue (the fitness brand association drives new streaming subscribers). At 15% operating margin, this adds $75M/year in incremental after-tax operating income (before tax effects).

## Real vs Nominal Cash Flows

### Consistency Rule

Cash flows and discount rates must be in the same terms:

| Cash Flows | Discount Rate | Correct? |
|-----------|--------------|----------|
| Nominal (include inflation) | Nominal WACC | Yes |
| Real (exclude inflation) | Real WACC | Yes |
| Nominal | Real WACC | No -- understates discount |
| Real | Nominal WACC | No -- overstates discount |

### Converting Between Real and Nominal

```
Nominal Rate = (1 + Real Rate) x (1 + Inflation Rate) - 1

Approximately: Nominal Rate = Real Rate + Inflation Rate
```

### Practical Recommendation

Use nominal cash flows with a nominal discount rate. This is the standard approach because:
- Revenue and cost projections are typically stated in nominal terms
- WACC components (risk-free rate, cost of debt) are already nominal
- Inflation effects on different cash flow components may differ (wages may grow faster than product prices)

## Depreciation and Tax Shields

### Depreciation Is Not a Cash Flow

Depreciation is a non-cash expense that reduces taxable income. It affects cash flow only through the tax shield:

```
Depreciation Tax Shield = Depreciation Expense x Marginal Tax Rate
```

### Computing After-Tax Operating Cash Flow

Two equivalent methods:

**Method 1 (Direct)**:
```
After-tax Operating CF = (Revenue - Cash Costs) x (1 - Tax Rate) + Depreciation x Tax Rate
```

**Method 2 (Indirect -- more common)**:
```
EBIT = Revenue - Cash Costs - Depreciation
After-tax Operating Income = EBIT x (1 - Tax Rate)
After-tax Operating CF = After-tax Operating Income + Depreciation
```

Both methods produce the same result. The depreciation tax shield is embedded in Method 2 because depreciation reduces EBIT and thus reduces taxes.

### Straight-Line vs Accelerated Depreciation

**Straight-line**: Equal depreciation each year.
```
Annual Depreciation = (Initial Cost - Salvage Value) / Useful Life
```

**Accelerated** (e.g., MACRS in the US): Higher depreciation in early years, lower in later years. Same total depreciation over the asset's life, but the front-loaded tax shields have higher present value.

All else equal, accelerated depreciation produces a higher NPV because the tax savings arrive sooner.

### Tax Rate: Marginal vs Effective

Use the **marginal tax rate** for project analysis:
- Marginal rate determines the tax impact of incremental cash flows
- Effective rate reflects past tax strategies and may not apply to future incremental decisions
- If the company has net operating losses (NOLs) that shelter income, the effective rate on the project may be lower temporarily, but revert to the marginal rate once NOLs are exhausted
